Marketing Experiments

I document small experiments to improve the product, usability, and communication. Each entry follows: Problem → Action → Result.

Experiment 1: Added an “About this project” link

Goal: Make the project context easy to find without distracting from the app.
Problem

The app is useful, but new visitors may not understand what it is for or why it exists. I needed a lightweight way to explain the project.

Action

I placed a small link (“About this project”) near the hero section, instead of adding another button.

Result
  • Clearer project context for first-time visitors
  • Kept the interface clean and calm (no extra buttons)

Note: I chose a link style to match the reading-journal atmosphere and avoid “button overload.”

Experiment 2: Supported both Japanese and English titles

Goal: Reduce friction when logging books with different title languages.
Problem

Japanese and English titles often differ, and some books are easier to manage using their original English titles. Without support for both, organizing entries becomes inconsistent.

Action

I added a way to keep Japanese and English titles separately, so users can record and search more reliably.

Result
  • More flexible logging (users can choose the most natural title)
  • Better consistency for search and organization

Experiment 3: Iterating the layout to improve focus

Goal: Help users understand the app quickly and start logging with confidence.
Problem

When the top section is too busy, users may not know where to start (what to click first).

Action

I kept the hero text short, grouped the value points, and aimed for a calm “reading note” feel.

Result
  • Cleaner first impression
  • Clearer path: read → understand → start logging

Next step: connect this with analytics to measure whether the change improves engagement.

Experiment 4: Simplified navigation (App link only on Home)

Goal: Keep the portfolio focused and reduce decision fatigue.
Problem

When “Try the App” appears on every page, visitors may jump around before understanding the project. This can weaken the portfolio story and make the experience feel inconsistent.

Action

I placed the “Try the App” link only on the Home page and kept other pages focused on their roles: Project (case overview), Experiments (iteration notes), and About (background).

Result
  • Clearer flow: Home → Project → Experiments
  • Less visual clutter and a calmer reading experience
  • A more consistent “case study” structure

Next step: measure whether this improves time on page and navigation depth (analytics).

Experiment 5: SEO optimization and README improvement

Goal: Improve search visibility and make the project easier to understand.
Problem

The original project title "Reading Journal" was simple but not descriptive. It did not clearly explain who the app was for or what it did. Some users may search for tools for Japanese learners rather than "reading journal".

The original README was also very minimal, which made it harder for visitors to quickly understand the project and its purpose.

Action

I updated the project title to:

Reading Journal for Japanese Learners – Track Books in Japanese & English

I also improved the README by adding:

  • A clearer project description
  • A feature list
  • A project concept explanation
  • A project goal section
Result
  • Clearer project positioning
  • Better search visibility potential
  • Easier understanding for new visitors
  • Stronger project presentation

Next step: Monitor search traffic and visitor behavior using analytics.

Experiment 6: Launched an Instagram account

Goal: Create a channel to share the project and reach potential users.
Problem

The Reading Journal app was publicly available, but there was no way to share updates or reach new users. Without a communication channel, it was difficult for people to discover the project.

Action

I created an Instagram account for the project and published:

  • An introductory post presenting the Reading Journal app
  • A story announcing upcoming mobile usability improvements

The link in the profile directs visitors to the app.
